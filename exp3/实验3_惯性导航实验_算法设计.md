## 🧩 Stage 2：核心算法

### 算法 1：双矢量法粗对准

**实际实现要点**:
- 单点取点 (t1, t2), 不用多组平均 (数据长度有限时反而降低精度)
- 0_0_0: t1=5s, t2=40s; 30_0_0: t1=5s, t2=70s
- 公式: `C_n^b = C_n^i(0) @ C_b^i(0)^T` (nav-to-body, 修正后)
- 欧拉角提取后解包裹: roll>90°时 roll-=180°, heading=-heading
- 仅需~40-70s数据, 不依赖全量

**实际结果** (0_0_0): Roll=0.039°, Pitch=-0.018°, Heading=0.798° (误差0.047° L2)
(30_0_0): Roll=0.063°, Pitch=0.017°, Heading=330.717° (误差0.102° L2)

### 算法 2：8维卡尔曼滤波精对准

```
状态: X = [φ_E, φ_N, δv_E, δv_N, ε_x, ε_y, ∇_x, ∇_y]  (8维)
量测: z = [v_E, v_N]  (仅水平速度, dim_z=2)

F矩阵分块 (8×8):
  F[0:2,0:2] = [[0, ω_U], [-ω_U, 0]]         地球自转耦合
  F[0:2,2:4] = I₂                             δv→φ
  F[0:2,4:6] = Cbn[0:2,0:2]                   ε→φ
  F[2:4,0:2] = [[0, f_U], [-f_U, 0]]         比力耦合 φ→δv
  F[2:4,6:8] = Cbn[1:3,0:2]                   ∇→δv

H = [0₂ₓ₂, I₂, 0₂ₓ₄]                          仅观测水平速度
```

**关键设计决策**:
- φ_U (航向误差) 静基座不可观测 → 移除, 航向锁定于粗对准
- δv_U (垂速) 由 fⁿ+gⁿ≈2g 主导 → 移除
- ε_z, ∇_z 不影响水平通道 → 移除
- 仅保留8个真正可观测的状态

**实际结果** (0_0_0): Roll=-0.003°, Pitch=-0.019°, Heading=0.798°, L2=0.028° (改善40%)

### 算法 3：姿态更新 (四元数递推)

**实际实现要点**:
- 陀螺补偿: `ω_nb^b = ω_ib^b - ε - C_n^b·ω_in^n` (零偏+地球自转)
- 初始姿态取自GPFPD首帧 (测试跟踪能力)
- 转序: 统一转为3-2-1与GPFPD对比 (DCM空间等价, 欧拉角数值因转序不同而异)
- GPFPD 3-2-1: Rx(roll) @ Ry(pitch) @ Rz(heading)

**实际结果** (纯惯性5-6分钟):
- 0_20_0: RMSE H=0.41° P=0.28° R=0.62°
- 0_0_90: RMSE H=0.53° P=0.09° R=0.19°
- -30_-20_180: RMSE H=0.04° P=1.31° R=0.65°
