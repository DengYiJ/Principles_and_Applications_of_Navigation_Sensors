## 1. SATXYZ2A 字段映射

### 文档规定

| 字段偏移 | 字段名 | 示例值 | 0-based索引 |
|:---:|:---|:---:|:---:|
| 1 | 系统 | GPS | 0 |
| 2 | PRN | 21 | 1 |
| 3 | X坐标 | -12946173.4100 | 2 |
| 4 | Y坐标 | 8286215.3673 | 3 |
| 5 | Z坐标 | 21665290.6048 | 4 |
| 6 | **卫星钟差** | 123616.744 | 5 |
| 7 | **电离层延迟** | 6.371 | 6 |
| 8 | **对流层延迟** | 2.559 | 7 |
| 9 | 保留1 | 0.0 | 8 |
| 10 | 保留2 | 0.0 | 9 |

**每个卫星块 = 14 个字段**（不是 10 个）

### 当前代码分析

```python
# visualization/plot_earth.py  parse_satxyz2a()
for j in range(0, len(fields) - 9, 10):   # ❌ 步长10，应为14
    block = fields[j:j + 10]               # ❌ 块大小10，应为14
    elev = float(block[6])   # ❌ 实际读的是"电离层延迟"6.371°
    azim = float(block[7])   # ❌ 实际读的是"对流层延迟"2.559°
```

**问题：** SATXYZ2A 中 **没有高度角和方位角字段**。之前代码错误地把电离层延迟（~6°）当作高度角、对流层延迟（~2.5°）当作方位角读取，导致 skyplot 上所有卫星高度角挤在 0~18°、方位角挤在北向。

### 当前代码分析

```python
# algorithms/extractor.py  extract_pseudorange_matrix()
RANGEA_BLOCK_SIZE: int = 10        # ❌ 应为11
for j in range(0, ..., 10):        # ❌ 步长10
    block = fields[j:j + 10]       # ❌ 块大小10
    prn = int(float(block[0]))     # ✅

```

**问题：** 伪距从 `block[2]` 改到 `block[3]`，信号掩码从 `block[9]` 改到 `block[10]`。
